# 📋 ЧЕК-ЛИСТ ЗАПУСКА МИРА

## Этап 1: Применение миграций

Выполните миграции в строгом порядке:

```bash
cd /workspace

# Применить миграцию структуры мира
mysql -u root -p DATABASE_NAME < database/021_massive_world_mechanics.sql

# Применить миграцию начальных данных
mysql -u root -p DATABASE_NAME < database/022_seed_world_data.sql
```

**Замените `DATABASE_NAME` на имя вашей базы данных.**

## Этап 2: Генерация карты

```bash
php scripts/generate_massive_world.php
```

- Размещено **8 убежищ** в безопасных зонах
- Созданы биомы: Горы (Запад), Пустошь (Центр), Леса (Север), Мексика (Юг), Братство (Восток)

## Этап 3: Настройка Cron (для погоды и существ)

Откройте crontab:
```bash
crontab -e
```

Добавьте строку для обновления мира каждые 5 минут:
```cron
*/5 * * * * php /workspace/scripts/update_world_state.php >> /var/log/fallout_world.log 2>&1
```

**Или для локальной разработки** запустите вручную:
```bash
php scripts/update_world_state.php
```

## Этап 4: Проверка результатов

### Проверка карты:
```sql
SELECT biome, COUNT(*) as count 
FROM map_nodes

```

### Проверка убежищ:
```sql
SELECT s.name, mn.x, mn.y, mn.biome 
FROM shelters s 
JOIN map_nodes mn ON s.map_node_id = mn.id;
```

## ⚠️ Возможные проблемы

### Ошибка "Table doesn't exist"
Убедитесь, что применены все предыдущие миграции (001-020).

### Ошибка "Duplicate entry" при генерации
Карта уже существует. Если нужно перегенерировать:
```sql
-- Внимание! Удалит все данные карты и позиции игроков!
TRUNCATE TABLE map_nodes;
TRUNCATE TABLE shelters;
```
Затем запустите генератор снова.

### Cron не работает
Проверьте путь к PHP:
```bash
which php
```
Используйте полный путь в crontab (например, `/usr/bin/php`).
