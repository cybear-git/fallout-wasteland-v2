```php
case 'logout':
    adminLogout();
    break;
```

## ⚡ ПРОИЗВОДИТЕЛЬНОСТЬ

### Анализ SQL-запросов:

#### ✅ Хорошо:
- Индексы на `pos_x, pos_y` для карты
- Prepared statements предотвращают N+1
- Пакетная вставка в генераторах (2000 за раз)
- Static PDO connection (`static $pdo`)

#### ⚠️ Проблемные места:

```sql
-- Проблема: SELECT * без LIMIT в некоторых местах
SELECT * FROM dungeons ORDER BY min_level DESC, name

-- Решение: Добавить LIMIT и выбрать только нужные колонки
SELECT id, dungeon_key, name, min_level, boss_key 
FROM dungeons 
ORDER BY min_level DESC, name 
LIMIT 50
```

#### Рекомендации:

1. **Пагинация списков** - уже добавлена переменная `$page`, `$limit = 50`
2. **Кэширование статистики** - дашборд кэшировать на 5 минут
3. **EXPLAIN для сложных запросов** - проверить запросы с JOIN
4. **Индексы** - добавить composite индексы:
   ```sql
   CREATE INDEX idx_admin_logs_created ON admin_logs(created_at DESC);
   ```
